Curve conventions

Which curve is being used?

Treasury benchmark

The Treasury curve is a liquid USD government benchmark and an approximate risk-free curve for rates-risk visualization. It is useful for Treasury risk, key-rate DV01 intuition, event-study visualization, and spread benchmarking.

SOFR/OIS distinction

Banks typically use SOFR/OIS curves for collateralized derivatives discounting. This project keeps SOFR/OIS as a future extension and does not claim production derivatives discounting.

Dirty Price

Cashflow pricing uses full price, not clean quote alone.

dirty_price = TDNOMPRC + TDACCINT P_dirty = P_clean + accrued_interest

Discount Factor

The demo uses continuously compounded zero rates for simple transparent discounting.

DF(T) = exp(-z(T) * T) z(T) = -ln(DF(T)) / T

Bond Pricing

A fixed-rate bond is the present value of coupons plus principal.

coupon_cashflow = face * coupon_rate / frequency P = sum_i cashflow_i * DF(T_i)

Yield To Maturity

Yield to maturity is the single constant yield that reproduces the dirty price from scheduled cashflows. CRSP TDYLD is used for observed YTM diagnostics; it is not a zero rate.

P_dirty = sum_i cashflow_i / (1 + y / frequency)^(frequency * T_i)

Zero Curve Fitting

Fitted zero rates are discount rates by maturity. A Treasury calibration fits them so model dirty prices approximately match observed dirty prices from selected issues.

min_z sum_j weight_j * (P_model_j(z) - P_market_j)^2

DV01

Positive DV01 means the position gains when rates fall by one basis point.

DV01 = (P_down_1bp - P_up_1bp) / 2

Key-Rate DV01

Key-rate DV01 bumps one curve pillar at a time to localize maturity exposure.

KRDV01_k = (P(node k down 1bp) - P(node k up 1bp)) / 2

Convexity

Convexity captures the second-order price response that DV01 misses for larger rate moves.

convexity = (P_down + P_up - 2 * P_0) / (P_0 * dy^2)

P&L Explain

Full revaluation is compared with first-order curve-risk attribution.

full_reval_pnl = PV_today - PV_yesterday rates_pnl_approx = sum_k KRDV01_k * (-delta_y_k_bp) residual = full_reval_pnl - rates_pnl_approx

Forward Rates

Forwards are implied rates between two zero-curve maturities.

f(T1,T2) = (z(T2) * T2 - z(T1) * T1) / (T2 - T1)

Curve Move In Basis Points

Basis-point curve moves make maturity-by-maturity shifts easy to scan.

delta_y_bp(T) = (z_today(T) - z_yesterday(T)) * 10000

Level, Slope, Curvature

Curve-factor diagnostics summarize broad shape changes.

level = average(z_2Y, z_5Y, z_10Y, z_30Y) slope = z_10Y - z_2Y curvature = 2 * z_5Y - z_2Y - z_10Y

3D Zero-Rate Surface

A 3D zero-rate surface plots rates by observation date and maturity so the event window can be read as a curve-shape movie in one static chart.

surface point = (valuation_date, maturity, z(date, maturity))

3D Forward-Rate Surface

The forward surface applies the forward-rate formula across dates and adjacent maturity intervals.

surface point = (valuation_date, start_maturity, f(T1,T2))

Swap Pricing

The demo keeps a simplified single-curve swap example. Real collateralized derivatives generally use SOFR/OIS discounting, which is a future extension.

PV_fixed = notional * fixed_rate * sum_i accrual_i * DF(T_i) PV_float = notional * (1 - DF(T_n)) PV_payer_fixed = PV_float - PV_fixed

Execution Analytics

The TCA module reports fill quality and simple transaction-cost measures for a deterministic hedge order.

VWAP = sum_i fill_price_i * fill_qty_i / sum_i fill_qty_i fill_ratio = filled_qty / order_qty slippage = average_fill_price - arrival_price shortfall = side_sign * (average_fill_price - arrival_price) * filled_qty

YTM versus zero rates

TDYLD is CRSP's promised daily yield and is useful for observed YTM diagnostics. It is not a zero rate. A fitted zero curve estimates discount rates that approximately reprice selected dirty prices and cashflows.

Limitations

The system is a simplified analytics engine, not a production valuation library. Calendars, market conventions, collateral discounting, spread risk, and robust curve bootstrapping are intentionally scoped as future extensions.
