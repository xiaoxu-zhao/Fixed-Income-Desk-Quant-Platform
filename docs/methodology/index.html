Model notes

Methodology

The project uses transparent fixed-income formulas so the generated dashboard can be audited from inputs to valuation, risk, P&L explain, and execution analytics.

Notation

t is valuation time, T is maturity, tau is year fraction, z(T) is the annual zero rate, DF(T) is the discount factor, and dy is a rate bump in decimal terms.

1. Data Conventions

The Phase 1 demo uses synthetic curves and positions. External Treasury files can be mapped from common CRSP-style fields such as coupon, maturity date, bid, ask, nominal price, accrued interest, yield, and duration.

mid_price = (bid + ask) / 2 dirty_price = clean_price + accrued_interest tau = days_between(valuation_date, cashflow_date) / 365.0

Rates are stored as decimals, so 4.25 percent is 0.0425. Treasury prices are quoted per 100 face value, while portfolio PV is reported in currency units after multiplying by face or notional.

2. Dirty Price

Clean price excludes accrued coupon. Dirty price is the cash amount used for settlement and valuation because it includes coupon earned since the previous coupon date.

P_dirty = P_clean + accrued_interest

3. Discount Factors And Zero Rates

The demo discounts cash flows with continuously compounded zero rates. The zero curve is interpolated by maturity.

DF(T) = exp(-z(T) * T) z(T) = -ln(DF(T)) / T

4. Bond Pricing

A fixed-rate bond is valued as the sum of discounted coupons plus discounted principal.

coupon_cashflow = face * coupon_rate / frequency P = sum_i cashflow_i * DF(T_i)

For portfolio reporting, instrument PV is multiplied by the position quantity.

5. Yield To Maturity

Yield to maturity is the single constant yield that reproduces the dirty price from the scheduled bond cash flows.

P_dirty = sum_i cashflow_i / (1 + y / frequency)^(frequency * T_i)

The solver searches for the yield y that makes model price equal observed dirty price.

6. Zero Curve Fitting

The current demo reads supplied zero-rate nodes. A Treasury calibration workflow would fit the curve by minimizing pricing errors across selected instruments.

min_z sum_j weight_j * (P_model_j(z) - P_market_j)^2

Short bills, notes, and bonds should be filtered for usable quotes, valid maturities, and consistent accrued-interest treatment before fitting.

7. Forward Rates

A forward rate is the implied rate between two maturities from the zero curve.

f(T1,T2) = (z(T2) * T2 - z(T1) * T1) / (T2 - T1)

Forward rates help separate carry, rolldown, and expected path information from the spot zero curve.

8. DV01

DV01 is the approximate change in value for a one basis point decrease in rates. The project uses the market convention that positive DV01 gains when rates fall.

DV01 = (P_down_1bp - P_up_1bp) / 2

9. Key-Rate DV01

Key-rate DV01 localizes rate sensitivity by bumping one curve pillar at a time, such as 2Y, 5Y, 10Y, or 30Y.

KRDV01_k = (P(curve with node k down 1bp) - P(curve with node k up 1bp)) / 2

The Phase 1 implementation uses nearest-pillar bumps rather than a production triangular bucket scheme.

10. Convexity

Convexity captures the second-order price response to rate moves.

convexity = (P_down + P_up - 2 * P_0) / (P_0 * dy^2)

It is useful when moves are large enough that DV01 alone misses curvature in the price-yield relationship.

11. P&L Explain

P&L explain compares full revaluation to first-order attribution from curve moves and risk sensitivities.

full_reval_pnl = PV_today - PV_yesterday rates_pnl_approx = sum_k KRDV01_k * (-delta_y_k_bp) residual = full_reval_pnl - rates_pnl_approx

The residual collects convexity, interpolation effects, simplified carry, and model approximation error.

12. Curve Move In Basis Points

Curve moves are shown in basis points so maturity-by-maturity changes are easy to scan.

delta_y_bp(T) = (z_today(T) - z_yesterday(T)) * 10000

13. Level, Slope, And Curvature

Simple curve factors summarize broad shape changes.

level = average(z_2Y, z_5Y, z_10Y, z_30Y) slope = z_10Y - z_2Y curvature = 2 * z_5Y - z_2Y - z_10Y

These factors are compact diagnostics, not a replacement for full key-rate risk.

14. 3D Zero-Rate Surface

A 3D zero-rate surface plots zero rates by observation date and maturity. It helps show how the curve evolves through time.

surface point = (valuation_date, maturity, z(valuation_date, maturity))

For sparse Treasury data, the surface should be built from fitted or interpolated zero curves on each observation date.

15. 3D Forward-Rate Surface

A forward-rate surface applies the forward-rate formula across dates and maturity intervals.

surface point = (valuation_date, start_maturity, f(T1,T2))

This view is useful for spotting changes in implied future rate paths.

16. Swap Pricing

The vanilla swap module uses a simplified single-curve framework. The fixed leg is discounted scheduled fixed coupons; the floating leg is approximated from discount factors.

PV_fixed = notional * fixed_rate * sum_i accrual_i * DF(T_i) PV_float = notional * (1 - DF(T_n)) PV_payer_fixed = PV_float - PV_fixed

Receiver-fixed swaps use the opposite sign.

17. Execution Analytics

The execution module reports simple transaction-cost analytics for a deterministic hedge order.

VWAP = sum_i fill_price_i * fill_qty_i / sum_i fill_qty_i fill_ratio = filled_qty / order_qty slippage = average_fill_price - arrival_price shortfall = side_sign * (average_fill_price - arrival_price) * filled_qty

For buys, side_sign is positive when paying more hurts performance. For sells, the sign is reversed.

18. Limitations

The project is an interview-ready simulation, not a production valuation stack. Important simplifications include synthetic data, simplified calendars and accruals, single-curve swap pricing, nearest-pillar key-rate bumps, stylized futures-to-yield mapping, and aggregate-depth execution replay.

Production extensions would add robust Treasury bootstrapping, OIS or SOFR discounting, curve instruments with market conventions, spread risk, stress testing, hedge optimization, and deeper market-data validation.
